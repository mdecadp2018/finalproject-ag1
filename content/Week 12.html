Week 12

【 Heroku 與 Github 同步衝突 】

●流程：

1、由組長所建立的 Heroku 的帳號

2、create 一個 app 名為 cadp2018-ag1，再將資料 clone 至近端

3、將小組資料夾內除 .git 資料夾以外資料複製過去(包含同步所需要的三個檔案)

4、接著 git add . 推送至 Heroku

按上述四個步驟即可完成  Heroku 與 Github 同步

●問題：

上述步驟結束後發生衝突判斷

1、可能為推送時間太接近，造成 Heroku 尚未和前次推送同步時又重複推送

2、指令 或是 clone  位址、檔案或推送次序有誤

●方法：

1、利用 git checkout 將小組倉儲版本退回同步以前

2、刪除現階段小組倉儲同步檔案，並再一次推送，後重新放入同步必須的三個檔案，並且覆蓋掉 Heroku 過去版本

3、重新建立 Heroku app 一切重新來過

後採用方法 2 ，順利解決
